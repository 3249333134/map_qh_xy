旅游相关 APP 功能及业务模式探讨
本次会议围绕旅游相关 APP，探讨平台优惠、API 费用、收费模式、成本承担及 APP 功能与服务器相关规划。
·平台优惠感知：认为美团优惠像提价后再优惠，不如京东优惠实在。
·API 相关：开发测试时 API 费用基本免费，运营按需购买，可整合费用到用户端，前期广告免费额度成本高。
·收费模式：探讨月付会员、终身会员、广告户解锁或付费点卡等收费方式。
·成本承担方案：最好由厂家或酒店承担 API 费用，通过返现当优惠，平衡成本。
·APP 功能规划：
·地图展示：以城市为主页，参考游戏导航，按街道划分区域大厅，通过广播实现区域内交互，接入地图显示社区内容。
            首页将默认以地图的形式展示，会获取个人的本地位置，会随着个人移动而改变，根据显示的卡尺大小确定显示内容
            地图上的位置可以点击，点击后会弹出详情页，点击详情页中的按钮会跳转到对应的页面，地图会显示热度与天气或者气候甚至其他信息，根据位置产生相应变化



·社区功能：打造社区基础，实现内容评论，设置侧栏、漂浮窗、小电视屏幕播报及弹幕滚动等，接入社区广播，展示店铺评论和推荐产品。
            
·攻略功能：提供攻略、攻略评论、攻略收藏等功能，用户可自主编辑攻略，通过攻略内容获取积分，可兑换商品。
            提供官方的攻略、插图、游玩推荐、社区高赞、景点由官方维护、内置活动与服务
·旅游规划：提供免费自由行功能辅助旅游，通过官方攻略、社群盈利，开发自定义路线、导入身体数据获取推荐等功能。
            分享的将以卡片的形式展示，类似于运动的跑图，点击将查看分享的内容，可以看到用户在整个更大的地图上显示的位置，以及期间的所见所闻
·引导模式：采用弱引导，到地点附近提示，实现震动反馈，用户自主打卡记录地点。
·内容整合：支持如小红书等内容复制自动规划，生成景点信息库，AI 调用攻略反馈给用户。
·共同回忆：考虑做共同编辑文档，整理成册发起众筹。
·服务器相关：讨论服务器节点，认为纯本地部署和纯云上两个模式较好，可创建服务器。

地图与APP相关规划及看法讨论
本次会议讨论地图设计、开发规划及对APP的看法，涉及界面风格、开发进度、功能优化及目标群体等。
·地图设计：
·界面风格：倾向简约、工具化风格，社交属性体现在播报交互，类似广播传递信号。
·区域划分与推送：按城市街区、景区景点划分区块，推送以距离进行点对点推送。
·界面展示：点开地图先切换到小一级位置，以卡片形式展示热点，放大可查看更详细热点；地图无操作时优先显示自己去过的路线。
·记忆卡片：点击地图标记弹出卡片，显示聊天窗（类似小弹幕）、基本信息、曾发照片（可拖时间维度）。
·不同端设计：桌面端和移动端显示有别，移动端需设置快速返回及类似目录功能，且窗口要有较大可扩展性。
·开发规划：5月找齐地图UI，开发需时间，联系御姐询问意见并整理计划书，逐步建立社群。
·APP看法：
·功能设计：觉得现有功能繁杂、易用性不够，应以卡片形式呈现服务、商品等内容，提高服务质量。
·目标群体：强调做APP要有优质群体
